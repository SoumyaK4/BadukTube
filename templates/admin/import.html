How it works
  • The import process will create new entries for topics, tags, ranks, lectures, and collections that don't exist.
  • Existing entries with the same name won't be duplicated.
  • All relationships between lectures, topics, tags, ranks, and collections will be preserved.
  • Paid collections status will be preserved.
  • For large datasets (over 500 lectures), the import may take several minutes.
  • The process is designed to handle large datasets efficiently with proper error recovery.
Large Dataset Recommendations
  • For very large datasets (1000+ lectures), consider breaking them into smaller exports by collection.
  • The import process is transactional per entity type, so even if it fails partway through, the successfully imported data will be preserved.
  • In case of errors, check the application logs for detailed information.
